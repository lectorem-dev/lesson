# Tech Fluency — учебная платформа для IT-английского

![TechFluencyLogo](frontend/public/techfluency.png)

## Краткое описание

Tech Fluency — MVP обучающей платформы для прохождения одного курса с уроками, тестами, прогрессом и PDF-сертификатом
после завершения обучения.

Проект создавался как учебно-инженерный кейс: задача заключалась в реализации работающей платформы и подготовке понятной
архитектуры, структуры репозитория, документации и материалов, по которым студент может разобраться в системе и
развивать ее дальше самостоятельно.

В рамках проекта автор выступал как разработчик, технический консультант и руководитель учебной реализации: помогал
сформировать MVP, выбрать стек, разложить систему на backend/frontend/content/infra, описать ключевые сценарии и
подготовить проект к демонстрации.

**Быстрый визуальный обзор:** [посмотреть скриншоты работы платформы](docs/demo/demo.md).

## Что реализовано

- Авторизация пользователей и разделение доступа по ролям.
- Административная панель управления пользователями.
- Кабинет студента с отображением курса и прогресса.
- Последовательное прохождение уроков.
- Markdown-уроки, импортируемые из `content/lessons/`.
- JSON-тесты, импортируемые из `content/tests/`.
- Учет прогресса прохождения уроков и тестов.
- Генерация PDF-сертификата после выполнения условий завершения курса.
- Docker-запуск PostgreSQL, backend и frontend.
- Документация для запуска, сопровождения, защиты и развития проекта.

## Архитектура

Проект разделен на несколько самостоятельных зон ответственности:

- `frontend/` — React/Vite приложение: маршрутизация, страницы, feature-модули, API-клиент и UI-компоненты.
- `backend/` — Python/FastAPI API: авторизация, пользователи, курс, прогресс, сертификаты, импорт контента и работа с
  базой данных.
- `PostgreSQL` — хранение пользователей, ролей, уроков, тестов и прогресса.
- `infra/` — Docker Compose для локального демонстрационного запуска.
- `content/` — учебные материалы курса: Markdown-уроки и JSON-тесты.
- `docs/` — техническая, учебная и презентационная документация.

## Быстрый запуск

### 1. Полный Docker-запуск для демонстрации

```powershell
cd infra
docker compose up --build
```

После старта:

- Frontend: http://localhost:3000
- Backend: http://localhost:8080
- Swagger: http://localhost:8080/docs
- ReDoc: http://localhost:8080/redoc

Демо-пользователь:

```text
login: admin
password: secret
```

### 2. Dev-запуск

Для разработки обычно запускаются:

- PostgreSQL в Docker через `infra/docker-compose.yml`;
- backend локально из `backend/`;
- frontend локально из `frontend/`.

Краткая схема:

```powershell
cd infra
docker compose up postgres
```

Затем backend запускается локально через Python/Uvicorn, а frontend — через Vite:

```powershell
cd frontend
npm install
npm run dev
```

Подробные команды для полного демонстрационного и dev-запуска описаны
в [руководстве локального запуска](docs/Руководство%20локального%20запуска.md).

## Как добавить новый контент

Учебный контент хранится отдельно от кода приложения:

- добавить Markdown-урок в `content/lessons/lesson-N.md`;
- добавить JSON-тест в `content/tests/test-N.json`;
- обновить `MANIFEST` в `backend/app/course/import_content.py`, если добавляется новый номер урока;
- перезапустить приложение, чтобы backend импортировал обновленный контент.

Подробная инструкция находится в
документе [Добавление контента на платформу](docs/Добавление%20контента%20на%20платформу.md).
