# Микро-отчет по стеку проекта

Проект представляет собой MVP обучающей платформы с frontend-приложением, backend API, PostgreSQL, импортом учебного
контента и генерацией PDF-сертификатов.

## Инфраструктура

Проект запускается через Docker Compose и состоит из трех сервисов:

| Сервис     | Назначение                                     |
|------------|------------------------------------------------|
| `postgres` | база данных PostgreSQL                         |
| `backend`  | FastAPI API, запуск миграций Alembic и Uvicorn |
| `frontend` | статическая сборка React-приложения под nginx  |

В Docker-сценарии frontend проксирует API-запросы `/api/*` на backend через nginx.

Основные переменные окружения:

| Переменная             | Назначение                          |
|------------------------|-------------------------------------|
| `COMPOSE_PROJECT_NAME` | имя Docker Compose проекта          |
| `POSTGRES_DB`          | имя базы данных                     |
| `POSTGRES_USER`        | пользователь PostgreSQL             |
| `POSTGRES_PASSWORD`    | пароль PostgreSQL                   |
| `POSTGRES_PORT`        | внешний порт PostgreSQL             |
| `BACKEND_PORT`         | внешний порт backend                |
| `FRONTEND_PORT`        | внешний порт frontend               |
| `JWT_SECRET`           | секрет подписи JWT                  |
| `VITE_API_BASE_URL`    | базовый URL API для frontend-сборки |

## Паттерны построения frontend

Frontend построен в слоистой структуре, близкой к feature-based / Feature-Sliced-подходу.

| Слой           | Назначение                                          |
|----------------|-----------------------------------------------------|
| `src/app`      | корневое приложение и роутинг                       |
| `src/pages`    | страницы: login, home, admin, student               |
| `src/features` | доменные фичи: auth, users, course, certificate     |
| `src/shared`   | общий HTTP-клиент, UI-компоненты, стили, env-конфиг |

Основные frontend-паттерны:

- Роутинг реализован через `createBrowserRouter`.
- Защита страниц вынесена в компонент `ProtectedRoute`.
- Доступ разделен по ролям `ADMIN` и `STUDENT`.
- Для каждой фичи есть отдельный API-модуль: например `courseApi.ts`, `authApi.ts`, `usersApi.ts`.
- Общий `axios`-клиент добавляет JWT в `Authorization` и обрабатывает ошибку `401`.
- JWT и краткий профиль пользователя хранятся в `localStorage`.
- Типы вынесены в слой `model`.
- Переиспользуемые UI-компоненты лежат в `shared/ui`.
- Markdown уроков рендерится на клиенте через `react-markdown` и `remark-gfm`.

## Паттерны построения backend

Backend построен модульно по доменам.

| Модуль        | Назначение                              |
|---------------|-----------------------------------------|
| `auth`        | логин и JWT                             |
| `users`       | пользователи, роли, админские операции  |
| `course`      | курс, уроки, тесты, импорт контента     |
| `progress`    | прогресс прохождения                    |
| `certificate` | генерация PDF-сертификата               |
| `core`        | конфигурация, безопасность, ошибки      |
| `common`      | общие исключения и ответы               |
| `db`          | SQLAlchemy engine, session и base model |

Основные backend-паттерны:

- Классический разрез `router -> service -> repository`.
- `router.py` отвечает за HTTP endpoint'ы.
- `service.py` содержит бизнес-логику.
- `repository.py` изолирует SQLAlchemy-запросы.
- Pydantic-схемы используются как DTO для request/response.
- SQLAlchemy ORM используется с `DeclarativeBase` и `Session`.
- Сессия БД прокидывается через FastAPI dependency `get_db`.
- Авторизация и проверка ролей реализованы через FastAPI dependencies: `get_current_user`, `require_roles`.
- Аутентификация stateless: JWT Bearer без серверной сессии.
- Пользователи не удаляются физически, а переводятся в статус `ARCHIVED`.
- При старте приложения через `lifespan` создается admin и импортируется контент курса.
- Контент курса валидируется строго: markdown-уроки и JSON-тесты проверяются при запуске.
- Ошибки централизованы через собственные exception-классы и регистрацию handlers.
- Миграции БД управляются Alembic.
- PDF-сертификат генерируется на backend через ReportLab после проверки завершения курса.
