# Локальный запуск проекта

Инструкция рассчитана на Windows, Git и терминал IDE PyCharm. Команды можно выполнять в обычном терминале PyCharm из
корня репозитория.

Проект состоит из трех частей:

- `backend` - Python/FastAPI API, порт `8080`.
- `frontend` - React/Vite интерфейс, dev-порт `5173`, Docker-порт `3000`.
- `postgres` - PostgreSQL, порт `5432`.

Демо-пользователь после старта backend:

```text
login: admin
password: secret
```

## 1. Демонстрация в Docker

Этот вариант поднимает все сразу в Docker: PostgreSQL, backend и frontend. Он удобен для демонстрации, потому что почти
не зависит от локального Python/Node.

Сначала проверьте, что Docker установлен и запущен:

```powershell
docker --version
docker compose version
```

Если команды не находятся или Docker не отвечает, установите Docker Desktop и запустите его.

Запуск всего проекта:

```powershell
cd infra
docker compose up --build
``` 

После успешного старта:

```text
Frontend: http://localhost:3000
Backend:  http://localhost:8080
Swagger:  http://localhost:8080/docs
ReDoc:    http://localhost:8080/redoc
```

Остановка:

```powershell
docker compose down
```

Если нужно остановить и удалить данные PostgreSQL:

```powershell
docker compose down -v
```

## 2. Подготовка к разработке

Для разработки backend и frontend запускаются локально, а PostgreSQL можно оставить в Docker.

### Проверить базовые инструменты

В терминале PyCharm из корня репозитория:

```powershell
git --version
docker --version
docker compose version
python --version
py --version
node --version
npm --version
```

Нужны:

- Docker Desktop с поддержкой `docker compose`.
- Python 3.10+; лучше Python 3.12. (если что `winget install -e --id Python.Python.3.12`)
- Node.js 20+; лучше Node.js 22 LTS.
- npm, который ставится вместе с Node.js.

Если Python не установлен, поставьте его с https://www.python.org/downloads/windows/. При установке отметьте
`Add python.exe to PATH`.

Если Node.js не установлен, поставьте LTS-версию с https://nodejs.org/.

После установки закройте и снова откройте терминал PyCharm, затем повторите проверку версий.

### Подготовить backend

Из корня репозитория:

```powershell
cd backend
```

Создать виртуальное окружение:

```powershell
py -3.12 -m venv .venv
```

Если Python 3.12 не установлен, используйте доступный Python:

```powershell
python -m venv .venv
```

Активировать окружение:

```powershell
.\.venv\Scripts\Activate.ps1
```

Если PowerShell запрещает запуск скриптов, выполните один раз:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

Затем снова активируйте окружение:

```powershell
.\.venv\Scripts\Activate.ps1
```

Обновить pip и поставить зависимости:

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Создать локальный `.env` для backend:

```powershell
Copy-Item .env.example .env
```

Проверьте, что в `backend\.env` есть такие значения:

```env
DATABASE_URL=postgresql+psycopg://lesson:lesson@localhost:5432/lesson
JWT_SECRET=change-this-secret-key
JWT_EXPIRATION_MINUTES=1440
CONTENT_DIR=../content
APP_PORT=8080
ADMIN_LOGIN=admin
ADMIN_PASSWORD=secret
```

Вернуться в корень репозитория:

```powershell
cd ..
```

### Подготовить frontend

Из корня репозитория:

```powershell
cd frontend
npm ci
```

npm ci --strict-ssl=false - на случай истока

Если `npm ci` завершился ошибкой из-за состояния `package-lock.json`, используйте:

```powershell
npm install
```

Создать локальный `.env.local`, чтобы frontend явно смотрел на локальный backend:

```powershell
Set-Content .env.local "VITE_API_BASE_URL=http://localhost:8080"
```

Вернуться в корень репозитория:

```powershell
cd ..
```

## 3. Запуск разработки

Для разработки удобно держать три терминала:

- Терминал 1: PostgreSQL в Docker.
- Терминал 2: backend локально.
- Терминал 3: frontend локально.

### Терминал 1: PostgreSQL

Из корня репозитория:

```powershell
cd infra
docker compose up -d postgres
```

Проверить, что контейнер запущен:

```powershell
docker compose ps
```

PostgreSQL будет доступен локально:

```text
host: localhost
port: 5432
database: lesson
user: lesson
password: lesson
```

### Терминал 2: backend

Из корня репозитория:

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
alembic upgrade head
uvicorn app.main:app --reload --host 127.0.0.1 --port 8080
```

После старта backend:

```text
API:     http://localhost:8080
Swagger: http://localhost:8080/docs
Health:  http://localhost:8080/api/health
```

При запуске backend автоматически создаст администратора и синхронизирует уроки из папки `content`.

### Терминал 3: frontend

Из корня репозитория:

```powershell
cd frontend
npm run dev
```

Vite покажет адрес в терминале. Обычно это:

```text
http://localhost:5173
```

Открывайте этот адрес в браузере для локальной разработки.

### Остановка разработки

Backend и frontend остановить в их терминалах:

```text
Ctrl+C
```

PostgreSQL остановить:

```powershell
cd infra
docker compose stop postgres
```

Если нужно удалить контейнер и сеть, но оставить volume с данными:

```powershell
docker compose down
```

Если нужно полностью сбросить базу данных:

```powershell
docker compose down -v
```

После сброса базы при следующем запуске backend снова выполните:

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
alembic upgrade head
```

## 4. Быстрые команды

Когда все настроенно и надо просто перезапустить после обновления кода.

### Терминал 1: PostgreSQL

```powershell
cd infra
docker compose up -d postgres
```

### Терминал 2: backend

```powershell
cd backend
source .venv/bin/activate
uvicorn app.main:app --reload --host 127.0.0.1 --port 8080
```

### Терминал 3: frontend

```powershell
cd frontend
npm install
npm run dev
```
